## 1. 추천 카테고리

- **1차:** IT·프로그래밍 > 자동화 (또는 "업무 자동화 / 워크플로우 자동화")
- **2차(서브):** IT·프로그래밍 > 챗봇·API 연동 / 프로그램 개발
- 크몽 카테고리 구조상 "업무 자동화" 또는 "API·연동개발" 메뉴가 노출되면 그쪽으로 등록. 없으면 "프로그램 개발 > 업무용 프로그램".

## 2. 상품 제목 후보 (검색 키워드 반영, 50자 내외)

1. **놓치는 리드 0건으로, 문의 들어오면 1분 안에 텔레그램 자동 알림 만들어 드립니다**
2. **반복 업무·수기 작업, Make(메이크) + API로 완전 자동화해 드립니다 (1일~ 구축)**
3. **광고 리드·문의 자동 수집 → 즉시 알림 → 문서 자동작성까지, 업무 자동화 구축**

> 메인 등록 제목은 1번 권장(성과 숫자 + 구체적 베네핏). 2번은 키워드("Make","자동화") 검색 대응용 A/B 후보.

## 4. 상세설명 본문 (복붙용)

### ▶ 혹시 이런 적 없으신가요?

광고는 돌아가는데, 정작 문의가 들어온 걸 **3~6시간 뒤에야** 확인합니다.
그 사이 고객은 이미 다른 곳에 연락했고, 들어온 리드는 엑셀 어딘가에 묻혀 사라집니다.
견적서, 보고서, 제안서는 매번 같은 양식을 **복사 → 수정 → 또 복사**. 하루 절반이 반복 작업으로 증발합니다.

**그 모든 반복을, 사람이 손대지 않아도 알아서 돌아가게 만들어 드립니다.**

---

### ▶ 이런 분께 딱 맞습니다

- 광고(메타/구글/네이버)로 들어오는 문의·리드를 **실시간으로 놓치지 않고** 받고 싶은 사장님
- 카톡·문자·이메일로 흩어진 문의를 **한 곳(텔레그램/슬랙)에 모아** 알림 받고 싶은 1인 사무소·소상공인
- 매번 똑같은 견적서·보고서·제안서를 **자동으로 채워** 만들고 싶은 분
- 직원 없이 혼자 돌리는데, **사람 한 명 몫의 단순 반복 업무**를 덜고 싶은 분
- "자동화 해보고 싶은데 뭘 써야 할지, 어디서부터 손대야 할지" 모르겠는 분

---

### ▶ Before / After — 무엇이 달라지나요

| | **Before (자동화 전)** | **After (자동화 후)** |
|---|---|---|
| 리드 확인 속도 | 3~6시간 뒤 확인 | **1분 이내 자동 알림** |
| 리드 누락 | 수기 관리 → 자주 놓침 | **누락 0건** (전수 기록) |
| 반복 문서작업 | 건당 4~8시간 | **10분 내외** (자동 생성) |
| 운영 인건비 | 사람이 계속 확인 | **사실상 무인 운영** |
| 월 운영비 | 별도 인력/툴 비용 | **월 $0 수준**으로 설계 가능 |

### ▶ 실제 구축 사례 (가공 없는 실측치)

**리드 기반 1인 사무소 — 메타 광고 → Make → 텔레그램 자동 알림 파이프라인**

- 구축 기간: **단 1일**
- 리드 확인 시간: 3~6시간 → **1분 이내**
- 리드 누락률: **0%** (들어온 문의 전수 자동 기록)
- 월 운영비: **$0** (무료 티어 범위 내 설계)
- 현재 상태: **지금도 실사용 중** — 일회성 납품이 아니라 현업에서 매일 돌아가는 시스템

> ※ 클라이언트 보호를 위해 상호·식별정보는 비공개로 합니다. "리드 기반 소상공인 / 1인 사무소" 수준으로만 안내드립니다.

**그 외 보유 역량**

- **문서 자동화 앱:** 기업상태표 입력 → Claude API → 사업계획서 + PPT 자동 생성. 문서작업 4~8시간 → **10분**.
- **AI 워크숍 운영:** 비개발자 20명 대상, 자동화 도구 **배포율 100%** 달성.

> ⚠️ 솔직하게 말씀드립니다. 위 "1일 구축"은 **검증된 실제 결과**이지만, 작업 난이도와 연동 환경에 따라 기간은 달라집니다. 표의 기간은 안전하게 잡은 기준이며, 첫 상담 때 고객님 상황에 맞춰 정확히 약속드립니다.

### ▶ 작업 프로세스 (이렇게 진행됩니다)

1. **무료 사전 상담 (당일~1일)** — 어떤 업무를 자동화할지, 현재 흐름을 듣고 자동화 가능 범위를 진단합니다. (구매 전 채팅 상담 환영)
2. **설계 & 견적 확정** — 자동화 시나리오 다이어그램으로 "무엇이 어떻게 돌아갈지"를 먼저 보여드립니다.
3. **구축** — Make / API / 채널 연동을 실제로 만들고, 테스트 데이터로 끝까지 돌려봅니다.
4. **검수 & 시연** — 화면 공유 또는 영상으로 동작을 함께 확인합니다.
5. **핸드오버** — 운영 가이드(문서/영상) 전달, 직접 켜고 끄고 관리하실 수 있게 인계합니다.
6. **무상 A/S** — 패키지별 기간 내 오류·미세 조정 무상 대응.

### ▶ 자주 묻는 질문 (FAQ)

**Q1. 개발 지식이 전혀 없는데 괜찮나요?**
네. 비개발자 20명 대상 워크숍에서 배포율 100%를 달성했습니다. 켜고 끄고 관리하는 법까지 영상·문서로 인계해 드립니다.

**Q2. 제 업무도 자동화가 되는지 모르겠어요.**
구매 전 채팅으로 현재 업무 흐름만 알려주세요. 자동화 가능 여부와 추천 패키지를 무료로 진단해 드립니다.

**Q3. 매달 비용이 계속 나가나요?**
가능한 한 무료 티어 범위에서 설계합니다. 실제 사례도 월 운영비 $0로 운영 중입니다. 트래픽이 큰 경우에만 API/툴 실비가 발생하며, 이 부분은 사전에 투명하게 안내합니다.

**Q4. 만들어주신 뒤에 고장 나면요?**
패키지별 무상 A/S 기간(7~30일) 내 오류·조정은 무료로 대응합니다. 이후에도 유지보수는 합리적인 비용으로 별도 지원합니다.

**Q5. Make 말고 다른 도구도 되나요?**
네. Make 외에 n8n, Zapier, 직접 코드(FastAPI 등) 어느 쪽이든 고객님 환경과 비용에 맞는 방식으로 구축합니다.

## 7. 첫 후기(리뷰) 확보 전략

후기가 0건인 신규 등록 상태에서는 전환이 가장 어렵습니다. 첫 3건의 후기를 빠르게 만드는 것이 최우선 목표입니다.

**전략:** STANDARD를 **한정 수량 "런칭 특가"**로 초기 1~2주만 운영하고(예: 정가 35만 → 런칭가 19~25만), 대신 **상세 후기 + 작업 결과 공유 동의**를 조건으로 겁니다. 동시에 지인·기존 클라이언트 네트워크에 "크몽 정식 후기 1건만 부탁"드려 실제 작업 기반의 진짜 후기를 확보합니다. 첫 후기에는 **Before/After 숫자(3~6시간→1분, 누락 0건)**가 드러나도록 유도해, 후기 자체가 다음 구매자의 신뢰 근거가 되게 만듭니다. 후기 3건·평점 5점이 쌓이면 런칭 특가를 종료하고 정가로 복귀합니다.
