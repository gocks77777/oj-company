> **상태:** MVP 개발 중 (~/ax-consulting/)

## 클라이언트 개요

| 항목 | 내용 |
|---|---|
| 업종 | AX(AI Transformation) 컨설팅 회사 |
| 규모 | 소규모 컨설팅 부티크 |
| 핵심 문제 | 컨설턴트 상담 후 문서 작성에 많은 시간 소요 |

---

## Before (진단)

**기존 워크플로우:**
```
1. 컨설턴트 → 고객 상담 (기업상태표 작성)
2. 기업상태표 → 다른 직원에게 전달
3. 직원이 수동으로 사업계획서 + PPT 작성
   → 소요 시간: 반나절~하루
```

**문제점:**
- 반복적인 문서 작업에 고숙련 인력 낭비
- 컨설턴트 1인당 처리 가능한 고객 수 제한
- 문서 품질이 담당자에 따라 편차 발생

---

## Solution (설계)

**자동화 아키텍처:**
```
기업상태표 입력 폼 (웹앱)
    ↓
Claude API — 사업계획서 생성 (구조화 프롬프트)
    ↓
python-pptx — PPT 자동 생성
    ↓
다운로드 / 검토 / 클라이언트 전달
```

**기술 스택:**
- Frontend: HTML/CSS/JS (심플 폼 UI)
- Backend: FastAPI (Python)
- AI: Claude API (Anthropic) — 사업계획서 본문 생성
- 문서: python-pptx (PPT 자동 생성)
- DB: SQLite (기업상태표 저장 이력)

**핵심 기능:**
1. 기업상태표 구조화 입력 폼 (업종, 현황, 목표, 강점, 약점 등)
2. Claude API로 맞춤형 사업계획서 초안 생성
3. 표준 PPT 템플릿에 자동 삽입 → 즉시 다운로드
4. 이력 관리 (고객별 문서 버전 관리)

## Target After (목표 성과)

| 지표 | Before | After (목표) |
|---|---|---|
| 문서 작성 시간 | 4~8시간 | 10분 이내 |
| 담당자 의존도 | 높음 | 낮음 (표준화) |
| 컨설턴트 처리 고객 수 | 제한적 | 2~3배 확대 |
| 문서 품질 일관성 | 편차 있음 | 표준화 |

## AX 인사이트

**이 프로젝트가 AX의 본질인 이유:**
1. **핵심 업무 프로세스 AI 내재화** — 지식 노동(문서 작성)을 AI로 대체
2. **컨설턴트 레버리지 확대** — 같은 인력으로 더 많은 고객 처리
3. **품질 표준화** — AI 기반 일관된 산출물
4. **도메인 특화 프롬프트 엔지니어링** — 범용 AI가 아닌 컨설팅 특화 설계

**확장 가능성:**
- 업종별 사업계획서 템플릿 다양화
- 자동 IR 자료, 투자 제안서 생성으로 확장
- SaaS 모델로 다른 컨설팅펌에 판매 가능
