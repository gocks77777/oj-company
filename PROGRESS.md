## 🎯 핵심 결정사항

- **4대 상품 → 3대로 정리** (로고송 제거: 단가 낮고 마진 나쁨)
  - A. 업무 자동화 (실적 있음, 검색수요 ↑)
  - B. **AI 직원팀 / 세컨드 브레인** ← 시그니처, 블루오션
  - C. 컨설팅펌 문서 자동화 (2억 IR 납품 실적)
- **과금**: 세팅비(1회) + 월 리테이너 하이브리드
- **채널 순서**: 크몽 등록(무료·자산) → 숨고(능동 영업, 메인) → 직접 영업
- **시그니처 엔진 = 헤르메스 에이전트** (오픈소스, 자체호스팅, 영구기억+스킬+멀티플랫폼+서브에이전트)

## 🧠 헤르메스 핵심 메모

- **무엇**: Nous Research 오픈소스 자율 AI 에이전트 (2026.2 출시, 신생).
- **능력**: 영구 기억(세컨드 브레인) / 셀프임프루빙 스킬(풀면 자동 저장, 다른 고객에 복제 가능 = O&J 해자) / 멀티플랫폼(디스코드·텔레그램·이메일) / 서브에이전트(AI 직원팀) / cron(24시간 무인) / 내장도구(웹검색·브라우저·이미지·TTS).
- **모델 연결(가성비)**: 하이브리드가 정답 — 싼 모델(DeepSeek/Kimi) 기본 + 어려운 것만 클로드.
  - ChatGPT Plus OAuth($20) = 깔끔, 단 주간 한도.
  - 클로드 = 구독 OAuth는 Max+추가크레딧+버그 → **API 키가 안전**.
  - 로컬 모델 = 아직 성능 부족, 시기상조.
- **한계(정직)**: 카카오톡·인스타 DM 자동연동 불가(텔레그램/이메일/웹챗만), 모델 API 비용 별도, 신생이라 불안정 → 내부 검증 후 판매.

## 👤 첫 고객 = 누나 (1호 베타 케이스)

- **사업**: 정책자금 컨설턴트 + 공인중개사 (병행) / 직원 4명
- **왜 1호**: 타깃 페르소나 그 자체 + 정해찬이 이미 자산 보유(`worker/` 통화→사업계획서 파이프라인, `funds_kb.json`, 부동산 리드 자동화 케이스)
- **목표 구성**: 디스코드 공유 두뇌 — 직원이 자료 올리면 업데이트, 누구나 물어보면 답하는 중앙 AI
- **세팅 방법**: ① 워크플로우 파악(진단 폼) ② 핵심 시드 주입 + 지속 캡처 ③ 자동화/스킬
- **세컨드 브레인 가치 = 기억 검색 아님 → "안 까먹기 + 24시간 실행"** (거절 대응 논리)

## ⚠️ 막힌 것 / 주의 (다음 세션 반복 방지)

- Web3Forms는 **서버(curl) 제출 차단** → 브라우저 폼만 작동. 테스트는 실제 브라우저로.
- 헤르메스 **카톡 자동연동 안 됨** → "카톡봇 됩니다"는 과장. 텔레그램/이메일로.
- 세컨드 브레인 **유료 고객 케이스 0** → 자사 도그푸딩으로 정직하게.
- 공개 리포라 **클라이언트 식별정보·창업자 의료정보(ADHD 등) 비공개** 원칙 유지.
